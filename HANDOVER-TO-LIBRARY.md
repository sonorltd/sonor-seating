# Seating Configurator ↔ Library — SSOT Handover

**From:** APP - Seating Configurator (`sonor-seating`)
**To:** APP - Library (owner of `furniture_catalogue`)
**Date:** 2026-07-17 · **Configurator target:** v0.2.0 (SSOT-driven rebuild)

The Seating Configurator is now a **read-only consumer** of the Library's furniture
SSOT. It never writes catalogue data. This doc is the contract: what the configurator
reads, the schema added to support it, and what the Library should maintain/fill.

---

## 1. What the configurator consumes (read-only)

| Table | Role for the configurator |
|-------|---------------------------|
| `furniture_ranges` | **The unit of choice.** One row per *manufacturer × model/style* — this is a "range". Drives the recommend step, constraint flags, materials, and "from" pricing. |
| `furniture_catalogue` | The items within a range (seats, armrests, accessories) with dimensions + WeQuote-schema pricing. Drives the configure step and the quote. |

The app loads both via the anon Supabase client with a 4-tier fallback
(Supabase → localStorage → bundled seed → inline). **No catalogue data is hardcoded
in the app.** The bundled seed is a generated snapshot, regenerated by
`data/build-seed.sh`.

---

## 2. Schema added to the Library (this handover created it)

### `furniture_catalogue` — new columns
- `range_id text` — FK-by-convention to `furniture_ranges.id`. Set by rule:
  `slug(supplier + '-' + metadata.model)` (e.g. `cineca-milan`, `cinelux-seating-kubrik`,
  `frontrow-seating-serenity`, `moovia-berlin`; suppliers with no model → `slug(supplier)`).
- `cost_price_gbp numeric` · `sell_price_gbp numeric` · `margin_pct numeric` —
  **WeQuote schema** (cost / sell / margin). `sell_price_gbp` is the customer-facing
  **MSRP**. Sourced from existing metadata: Cineca ← `seating_prices` (trade/SRP),
  Cinelux ← `metadata.list_price_gbp_incvat`, FrontRow ← `metadata.variants.SGL-FAB.{cost,sell}`.

### `furniture_ranges` — new table (Library-owned)
```
id (slug) · manufacturer · name (model/style) · style · description
hero_img · thumb_img · product_url
capability jsonb   -- recommend + constraint attributes (see §3)
pricing_from jsonb -- {cost, sell, incl_vat, unit:'per seat'} cheapest seat in range
materials jsonb    -- upholstery options [{id,name,group,tier,swatch,colours:[{name,hex}],upcharge_pct,note}]
finishes jsonb     -- [{id,label,note}]
sort_order · enabled · metadata jsonb · created_at · updated_at
```
RLS: anon SELECT/INSERT/UPDATE (no DELETE), matching the Library convention.

## 3. The capability contract (how recommendations + flags work)

`furniture_ranges.capability` is what makes "recommend ranges for this layout" and
"flag: no daybed in this range" possible. The configurator reads:

- `seat_width_min_mm` / `seat_width_max_mm` — to check a row of N seats fits the room width.
- `min_seats` / `max_seats` / `default_seats_per_row` (Cineca) — layout limits.
- `motor_options[]` — e.g. `["fixed","1motor","2motor"]`.
- `reclined_depth_mm` — depth/row-gap checks.
- `config_options[]` — e.g. Cinelux `["Single","Two",…]`.
- `features{}` — booleans: `chaise, daybed, loveseat, sofa, corner, ottoman, modular, reclining`.
  **These drive the constraint flags.** If the customer asks for a daybed and
  `features.daybed=false`, the range is flagged "no daybed option in this range".
- `metadata.seating_range` (bool) — the app only offers ranges where this is true
  (excludes accessory/storage/blind suppliers like Great Northern Popcorn, Middle
  Atlantic, Silent Gliss).
- `metadata.priced` (bool) — false ⇒ the app shows "MSRP on request" instead of a price.

## 5. Requests to the Library (to make this fully customer-facing)

1. **Fill MSRP** for Moovia / Fortress / Palliser / Cineak — via scrape or price-list
   upload, into `furniture_catalogue.sell_price_gbp` (+ cost/margin). The app already
   shows "MSRP on request" until then.
2. **Model per-range upholstery options** (materials + colourways) for non-Cineca ranges,
   into `furniture_ranges.materials` (same shape as Cineca), so the configure step offers
   real fabric/leather + colour choices instead of a single `finish` string.
3. **Confirm `capability.features` + dimensions** for `_needs_review` ranges (esp. seat
   widths, motor options, daybed/chaise flags) — these drive recommendations, so wrong
   flags = wrong advice.
4. **Own it going forward.** `furniture_ranges` + the pricing columns are Library-owned
   SSOT; the Library UI should gain edit surfaces for them (like the shades/finishes
   registries). The configurator will only ever read.

## 6. Reciprocal
The configurator regenerates its offline seed from these tables (`data/build-seed.sh`);
whenever the Library changes furniture data, re-run it (or the nightly seed task) so the
offline tier stays fresh. Trade pricing (`cost_price_gbp`) is carried but hidden — a
trade toggle is deferred; MSRP (`sell_price_gbp`) is the only price shown for now.
